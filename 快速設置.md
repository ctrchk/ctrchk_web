# 快速設置（3 步驟解決登入問題）

> 更新日期：2026-03-07  
> 適用版本：已改用 `pg` 套件連接 Neon + Google Identity Services (GIS)

---

## 您遇到的問題

❌ **註冊失敗**：「資料庫尚未設置，請聯絡管理員」  
❌ **Google 登入失敗**：「Supabase auth 方法不可用」

## 原因（已修正）

| 問題 | 舊版做法 | 新版做法 |
|------|---------|---------|
| 資料庫連接 | `@vercel/postgres`（需 Vercel 原生整合）| `pg` 套件（支援任何 Postgres，包括直接 Neon）|
| Google 登入 | Supabase Google OAuth（設定複雜）| Google Identity Services GIS（直接、簡單）|

---

## 解決方法（3 個步驟）

---

### 📦 步驟 1：建立 Neon 資料庫

> 如果您已在 Neon 建立資料庫並執行了 SQL，可直接跳到步驟 2。

1. 前往 [Neon Console](https://console.neon.tech/)
2. 建立新專案（例如 `ctrchk`），選擇區域（Singapore 或 US East）
3. 建立完成後，點擊 **SQL Editor**
4. 貼入並執行以下 SQL：

```sql
CREATE TABLE IF NOT EXISTS users (
    id SERIAL PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255),
    user_role VARCHAR(20) DEFAULT 'junior',
    full_name VARCHAR(100),
    phone VARCHAR(20),
    birthdate DATE,
    experience VARCHAR(20),
    bike_type VARCHAR(20),
    preferred_area VARCHAR(50),
    profile_completed BOOLEAN DEFAULT FALSE,
    profile_completion_date TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW(),
    auth_provider VARCHAR(20) DEFAULT 'email',
    google_id VARCHAR(255)
);

CREATE INDEX IF NOT EXISTS idx_users_email ON users(email);
CREATE INDEX IF NOT EXISTS idx_users_google_id ON users(google_id);

CREATE TABLE IF NOT EXISTS cycling_history (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
    ride_date DATE NOT NULL,
    distance_km DECIMAL(10, 2),
    route_name VARCHAR(255),
    created_at TIMESTAMP DEFAULT NOW()
);
```

5. 複製您的 **Connection String**（格式如 `postgresql://user:password@host/dbname?sslmode=require`）

---

### 🔐 步驟 2：在 Vercel 設置環境變數

1. 登入 [Vercel Dashboard](https://vercel.com/dashboard)
2. 選擇您的專案 → **Settings** → **Environment Variables**
3. 逐一點擊 **Add New** 並填入以下三個變數：

| 名稱 | 說明 | 範例 |
|------|------|------|
| `DATABASE_URL` | Neon 連接字串（含密碼）| `postgresql://user:pass@host/db?sslmode=require` |
| `JWT_SECRET` | 隨機長字串（最少 32 字元）| 可用 [random.org](https://www.random.org/strings/) 生成 |
| `GOOGLE_CLIENT_ID` | Google OAuth 用戶端 ID（步驟 3 取得）| `123456789-abc.apps.googleusercontent.com` |

4. 每個變數的 **環境** 選擇：Production ✅、Preview ✅、Development ✅
5. 儲存後，前往 **Deployments** → 點最新部署的 **...** → **Redeploy** ✅

---

### 🔵 步驟 3：設置 Google 登入（Google Identity Services）

> ⚠️ 舊版的 Google+ Sign-In 已於 2019 年停用。本網站現已改用最新的 **Google Identity Services (GIS)**，設定更簡單。

#### 3.1 建立 Google OAuth 憑證

1. 前往 [Google Cloud Console](https://console.cloud.google.com/)
2. 頂部下拉選單 → **建立新專案**（例如 `CTRC HK`）→ 等待建立完成
3. 左側選單：**APIs & Services** → **OAuth consent screen**
   - 選擇 **External**（公開）→ **Create**
   - App name：`CTRC HK`
   - User support email：您的 Google 帳號
   - 往下滾動 → **Save and Continue**（其他欄位可略過）
   - 重複 **Save and Continue** 直到完成
4. 左側選單：**Credentials** → **Create Credentials** → **OAuth 2.0 Client ID**
5. Application type：**Web application**
6. Name：`CTRC HK Web`
7. **Authorized JavaScript origins**（授權來源）添加：
   ```
   https://www.ctrchk.com
   ```
   （如果還在測試，也可添加 `http://localhost:3000`）
8. **Authorized redirect URIs**（本版本的 GIS **不需要**填寫此項）
9. 點擊 **Create** → 複製 **Client ID**（格式如 `123456789-abc.apps.googleusercontent.com`）

#### 3.2 將 Client ID 填入 Vercel

1. 回到 Vercel Dashboard → Settings → Environment Variables
2. 將剛才複製的 **Client ID** 填入 `GOOGLE_CLIENT_ID` 環境變數
3. 重新部署（Redeploy）✅

## ❓ 常見問題

### 問題 1：點擊「使用 Google 登入」沒有出現 Google 按鈕
- 檢查：`GOOGLE_CLIENT_ID` 環境變數是否已設置並重新部署
- 檢查：Google Cloud Console → OAuth consent screen 是否已完成填寫
- 注意：按鈕由 Google 渲染，需要數秒才會出現

### 問題 2：Google 登入時出現「Error 400: redirect_uri_mismatch」
- 檢查：Google Cloud Console → Credentials → OAuth 2.0 Client ID → Authorized JavaScript origins
- 確保已添加 `https://www.ctrchk.com`（不含尾斜線）

### 問題 3：電郵註冊時仍出現「資料庫尚未設置」
- 檢查：`DATABASE_URL` 環境變數是否正確（複製整個 Neon 連接字串）
- 到 Neon SQL Editor 執行：`SELECT * FROM users LIMIT 1;` 確認資料表存在

### 問題 4：設定後沒有變化
- 確認已重新部署（步驟 2.5）
- 清除瀏覽器快取（Ctrl+Shift+R 或 Cmd+Shift+R）並重新整理頁面
