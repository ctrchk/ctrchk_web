## v1.5.2.3 — 2026-03-25

### 修復 (Bug Fixes)
- **聊天訊息氣泡文字換行修復** — 修復聊天訊息每個字元各佔一行的排版問題；根本原因：`.msg-bubble` 的 `max-width: 70%` 是相對其無明確寬度的父 `div` 計算，形成循環依賴，瀏覽器解析後幾乎為零寬度；已將 `max-width` 移至新增的 `.msg-inner` flex 子項層級，使百分比相對 `.msg-bubble-wrap` 的寬度正確計算，並補充 `overflow-wrap: break-word` 提升相容性
- **聊天切換用戶延遲修復** — 修復切換對話時停留在舊用戶畫面數秒的問題；新增記憶體快取（`msgCache`，以 `peerId` 為鍵）：切換對話時立即從快取渲染上次訊息，同時在背景更新最新資料；若使用者在回應抵達前已切換至其他對話，舊回應將被捨棄，避免訊息錯亂

## v1.5.2.1 — 2026-03-25

### 修復 (Bug Fixes)
- **聊天頁手機版全面修復** — 修復手機版聊天功能多個核心故障：
  - **對話列表點擊無反應** — 根本原因：`renderConvList` 以 `JSON.stringify` 直接插入 `onclick` HTML 屬性，雙引號破壞屬性值導致事件失效；已將輸出字串的雙引號改以 `&quot;` HTML 實體編碼
  - **新對話用戶搜尋無反應** — 同上，`renderUserResults` 中相同問題已一併修復
  - **對話搜尋框無反應** — `filterConvList` 未暴露至全域（`window`），導致 `oninput` 處理器呼叫失敗；已加入 `window.filterConvList = filterConvList`
  - **關閉對話列表後無法復原** — 舊版 CSS 錯誤地將 `btn-new-chat` 類別在所有手機介面下隱藏，致使「查看對話」及「新對話」捷徑按鈕消失；已改為全螢幕滑入式面板設計：對話列表以滑動方式全螢幕展示，不再依賴覆蓋層定位，消除 z-index 衝突
  - **漢堡選單（三橫線）按下後無法返回** — 已於手機版新增返回按鈕（訊息區頂部），並確保空白狀態的捷徑按鈕正常顯示

## v1.0.1 — 2026-03-21

### 修復 (Bug Fixes)
- **電郵驗證**：修復新用戶完成註冊後未能收到驗證電郵的問題。  
  根本原因：`api/register.js` 原本以非阻塞（fire-and-forget）方式發送驗證郵件，在 Vercel serverless 環境中，系統會在回傳回應後立即終止函式執行，導致尚未完成的 SMTP 連線被中斷。  
  解決方法：改為 `await` 等待郵件發送完成後再回傳回應，確保郵件能成功送出。
