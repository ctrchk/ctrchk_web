## v2.0.6 Beta — 2026-06-11

### 新功能 (New Features)
- **單車路線圖籍 (Route Catalog) 升級** — 在騎行頁面 (`ride.html`) 整合圖籍入口，支援在騎行過程中實時查看進度、玩法說明及區域征服狀態。集齊可領取 **10,000 XP** 及 **1,000 里程幣**，並自動解鎖所有特別篇路線。
- **網頁版視覺優化** — 網頁版路線列表不再套用「未征服灰階濾鏡」，確保非 PWA 用戶也能看到豐富色彩的路線資訊。

### 修復 (Bug Fixes)
- **自由模式功能修復** — 修復 `ride.html` 自由模式 UI 缺失、計時器停止及無法保存行程的嚴重 Bug；優化自由模式結束時的路線相似度匹配算法。
- **環境感知門控** — 徹底隱藏網頁版內的「自由模式」及「開始騎行」入口，引導用戶使用 PWA 體驗核心騎行功能。
- **地圖圖層恢復** — 恢復導航頁面 (`nav.html`) 的 **3D 建築物 (金卡專屬)** 及 **單車泊位 (銀卡/金卡)** 顯示，修正 Mapbox 資料源識別邏輯。

## v2.0.4 Beta — 2026-06-05

### 新功能 (New Features)
- **自由騎行模式 (Free Ride Mode)** — `ride.html` 新增自由騎行功能，含 3-2-1 倒數計時。系統會自動偵測沿途所有站點並給予 XP，結束時自動匹配相似路線發放獎勵。
- **好友系統 (Friend System)** — `chat.html` 及 `api/user.js` 新增好友功能，支援搜尋用戶、發送/接受好友請求，並可查看好友的等級、里程等狀態。
- **螢幕常亮 (Screen Wake Lock)** — 騎行及導航期間自動啟用 Wake Lock API，防止螢幕自動關閉。
- **快取優化 (Initialization & Caching)** — `index.html` 新增「跳過載入」按鈕；`js/main.js` 實現全域路線及資料快取，大幅提升頁面切換速度。
- **主頁連戰顯示** — `index.html` 用戶卡片新增「連戰」天數顯示。
- **途經點 UI 升級** — `nav.html` 導航途經點功能現對所有用戶可見，銅卡用戶會顯示鎖定圖示及升級提示。

### 修復 (Bug Fixes)
- **定位邏輯修正** — 修正天氣、導航及騎行頁面的自動定位邏輯。現在僅在 App (PWA) 模式下自動要求權限，網頁版改為手動觸發，解決網頁版強制取位及 App 內失效問題。
- **導航地圖跟隨修正** — 修正導航時地圖強制回中的問題。新增手動操作偵測，用戶縮放或平移地圖後會暫停自動跟隨，點擊指南針可恢復。
- **銀卡停車系統修復** — 修正 `nav.html` 中單車泊位圖層的檔案路徑，恢復銀卡用戶的停車位顯示功能。
- **金卡主題相容性修正** — 修正 `profile.html` 中金卡用戶關閉黑金模式後無法切換亮色模式的問題。

## v1.5.2.1 — 2026-03-25

### 修復 (Bug Fixes)
- **聊天頁手機版全面修復** — 修復手機版聊天功能多個核心故障：
  - **對話列表點擊無反應** — 根本原因：`renderConvList` 以 `JSON.stringify` 直接插入 `onclick` HTML 屬性，雙引號破壞屬性值導致事件失效；已將輸出字串的雙引號改以 `&quot;` HTML 實體編碼
  - **新對話用戶搜尋無反應** — 同上，`renderUserResults` 中相同問題已一併修復
  - **對話搜尋框無反應** — `filterConvList` 未暴露至全域（`window`），導致 `oninput` 處理器呼叫失敗；已加入 `window.filterConvList = filterConvList`
  - **關閉對話列表後無法復原** — 舊版 CSS 錯誤地將 `btn-new-chat` 類別在所有手機介面下隱藏，致使「查看對話」及「新對話」捷徑按鈕消失；已改為全螢幕滑入式面板設計：對話列表以滑動方式全螢幕展示，不再依賴覆蓋層定位，消除 z-index 衝突
  - **漢堡選單（三橫線）按下後無法返回** — 已於手機版新增返回按鈕（訊息區頂部），並確保空白狀態的捷徑按鈕正常顯示

## v1.0.1 — 2026-03-21

### 修復 (Bug Fixes)
- **電郵驗證**：修復新用戶完成註冊後未能收到驗證電郵的問題。  
  根本原因：`api/register.js` 原本以非阻塞（fire-and-forget）方式發送驗證郵件，在 Vercel serverless 環境中，系統會在回傳回應後立即終止函式執行，導致尚未完成的 SMTP 連線被中斷。  
  解決方法：改為 `await` 等待郵件發送完成後再回傳回應，確保郵件能成功送出。
